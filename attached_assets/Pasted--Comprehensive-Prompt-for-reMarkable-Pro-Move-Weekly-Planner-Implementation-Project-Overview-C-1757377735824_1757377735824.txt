# Comprehensive Prompt for reMarkable Pro Move Weekly Planner Implementation

## Project Overview

Create an optimized 8-page PDF weekly planner for the reMarkable Pro Move tablet that features bi-directional linking between a weekly overview and individual daily pages. The planner must be optimized for e-ink display, stylus interaction, and efficient navigation.

## Technical Requirements

### Page Structure

- **Page 1**: Weekly overview grid (Monday-Sunday)
- **Pages 2-8**: Individual daily pages (Monday through Sunday)
- **Paper size**: reMarkable Pro Move dimensions (229.5 x 297mm / Letter size compatible)
- **Export format**: PDF with interactive links

### E-ink Optimization

1. **Contrast**: Pure black (#000000) on white (#FFFFFF)
1. **Grays**: Maximum 5% gray for organizational elements only
1. **Line weights**: Minimum 0.5pt, maximum 2pt
1. **Anti-aliasing**: Disabled for crisp lines
1. **Fonts**: Sans-serif only, minimum 10pt size

### Button Specifications

- **Size**: Minimum 10mm x 10mm tap target
- **Border**: 2pt black border, white fill
- **Padding**: 3mm internal padding
- **Spacing**: 5mm between buttons
- **Hover state**: Not required (e-ink limitation)
- **Active state**: Bold border (3pt) when selected

## Implementation Notes

### PDF Generation

1. Use a headless browser (Puppeteer, wkhtmltopdf) to convert HTML to PDF
1. Ensure PDF links are preserved during conversion
1. Set PDF page size to Letter/A4 for reMarkable compatibility
1. Enable JavaScript for link generation if using dynamic content

### Link Implementation

For PDF interactivity, use one of these methods:

1. **HTML anchors**: Convert to PDF internal links
1. **JavaScript**: Add event listeners before PDF generation
1. **PDF libraries**: Use PDFKit or similar to add link annotations post-generation
